# 🤖 بوت إدارة نقاط المشرفين

بوت ديسكورد احترافي لإدارة نقاط المشرفين،
يضم 7 أنظمة متكاملة مع واجهة تفاعلية كاملة بالـ Slash Commands.

## 🚀 خطوات التشغيل

### 1 — إنشاء البوت في Discord

1. اذهب إلى [discord.com/developers/applications](https://discord.com/developers/applications)
2. اضغط **New Application** وسمّ البوت
3. من القائمة الجانبية اختر **Bot**
4. اضغط **Reset Token** وانسخ التوكن
5. فعّل هذه الـ Intents:
   - ✅ Server Members Intent
   - ✅ Message Content Intent
6. من **General Information** انسخ **Application ID**

### 2 — دعوة البوت للسيرفر

استخدم هذا الرابط (بدّل CLIENT_ID بالـ ID الخاص بك):
```
https://discord.com/oauth2/authorize?client_id=CLIENT_ID&permissions=1099511696390&scope=bot+applications.commands
```

الصلاحيات المطلوبة تشمل:
- Read Messages / View Channels
- Send Messages + Embed Links
- Manage Roles
- Moderate Members (Timeout)
- Read Message History

### 3 — إعداد الملفات
```bash
# استنساخ أو تنزيل المشروع
cd modbot

# تثبيت المكتبات
npm install

# إنشاء ملف الإعدادات
cp config.example.js config.js
```

افتح `config.js` وضع:
```js
export default {
  token: "توكن_البوت",
  clientId: "Application_ID",
};
```

### 4 — تسجيل الأوامر (مرة واحدة فقط)
```bash
npm run deploy
```

ستظهر رسالة: `✅ تم تسجيل جميع الأوامر بنجاح!`

> إذا عدّلت أسماء الأوامر لاحقاً أعد تشغيل هذا الأمر.

### 5 — تشغيل البوت
```bash
# تشغيل عادي
npm start

# تشغيل مع إعادة تحميل تلقائية عند التعديل (للتطوير)
npm run dev
```

## 📋 الأوامر الكاملة

### 📊 النقاط

| الأمر | الوصف | الصلاحية المطلوبة |
|-------|-------|-------------------|
| `/add [reason]` | إضافة نقاط لمشرف أو أكثر | إدارة |
| `/remove [reason]` | خصم نقاط من مشرف أو أكثر | إدارة |
| `/points [member]` | عرض نقاط مشرف | مشرف |
| `/top` | لوحة الصدارة (10 في كل صفحة) | مشرف |
| `/reset [member]` | تصفير نقاط مشرف أو الكل | إدارة |

### 🎖 الترقيات

| الأمر | الوصف | الصلاحية المطلوبة |
|-------|-------|-------------------|
| `/rank [member]` | بطاقة رتبة المشرف مع progress bar | مشرف |
| `/promote up @member reason` | ترقية يدوية للمرحلة التالية | إدارة |
| `/promote down @member reason` | تخفيض يدوي للمرحلة السابقة | إدارة |
| `/promote info` | عرض السلم الوظيفي كاملاً | الكل |

### ⚖️ الموديريشن

| الأمر | الوصف | الصلاحية المطلوبة |
|-------|-------|-------------------|
| `/warn @member reason` | تحذير عضو (يُرسل للمراجعة) | إدارة |
| `/timeout @member duration reason` | توقيف عضو (يُرسل للمراجعة) | إدارة |
| `/appeal review_id` | استئناف على عقوبة مرفوضة | مشرف |
| `/memberlog @member` | سجل عقوبات عضو كامل | إدارة |

### 📋 المهام

| الأمر | الوصف | الصلاحية المطلوبة |
|-------|-------|-------------------|
| `/mytasks` | عرض مهامك الحالية مع progress bar | مشرف |
| `/task setup role type goal period points` | إعداد مهمة لرتبة | إدارة |
| `/task list` | قائمة كل المهام المُعدَّة | إدارة |
| `/task remove role` | حذف مهمة رتبة | إدارة |

### ⚙️ الإعدادات

| الأمر | الوصف | الصلاحية المطلوبة |
|-------|-------|-------------------|
| `/setup` | لوحة الإعدادات الكاملة | إدارة |
| `/help` | عرض جميع الأوامر | الكل |

## 🛡 نظام Anti-Abuse

| الحماية | الآلية |
|---------|--------|
| منع الإضافة لنفسه | فحص `userId === executorId` في /add و /remove |
| حد أقصى للنقاط | `maxAdd` و `maxRemove` من الإعدادات |
| cooldown بين الأوامر | `abuseCooldown` بالثواني لكل مستخدم |
| حد أقصى للأعضاء | `maxMembers` في كل عملية |
| منع النقاط تحت الصفر | `Math.min(amount, current.total)` في /remove |
| حد يومي XP | `dailyLimit` في إعدادات الـ XP |
| cooldown XP | `cooldown` بالثواني بين كل رسالة |

## 🐛 حل المشاكل الشائعة

**البوت لا يرد على الأوامر:**
- تحقق أن `npm run deploy` نجح
- تحقق من الـ Intents في Developer Portal
- تحقق من صلاحيات البوت في القناة

**`Missing Permissions` عند التايم أوت:**
- الرتبة التي تديرها يجب أن تكون أعلى من رتبة العضو المستهدف في هرمية الرتب
- يجب أن تكون رتبة البوت أعلى من رتبة العضو المستهدف

**الأوامر لا تظهر لبعض الأعضاء:**
- الأعضاء بدون رتبة مشرف أو إدارة محقهم يرون فقط: `/points` `/top` `/rank` `/help`
- هذا صحيح وهو المقصود من نظام الصلاحيات

**XP لا يُمنح:**
- تحقق أن للعضو رتبة مشرف في `/setup`
- تحقق أن `Message Content Intent` مفعّل في Developer Portal

## 🏆 نظام الترقيات — شرح تفصيلي

### إعداد المراحل
من `/setup` → 🏆 تعديل مراحل المكافآت، تُدخل حتى 5 مراحل:
```
500  123456789012345678    ← 500 نقطة = رتبة ID هذا
1000 234567890123456789
2500 345678901234567890
5000 456789012345678901
10000 567890123456789012
```

### آلية العمل
- بعد **كل** تغيير في النقاط يُستدعى `checkPromotion()` تلقائياً
- النظام **لا يخفّض** أحداً تلقائياً أبداً
- التخفيض اليدوي فقط عبر `/promote down`
- عند الترقية: إعلان في القناة + DM للمشرف + تسجيل في التاريخ

## 📋 نظام المهام — شرح تفصيلي

### أنواع المهام
| النوع | يتتبع |
|-------|-------|
| `messages` | عدد الرسائل المُرسلة |
| `moderation` | عدد العقوبات المقبولة |
| `voice` | ساعات الفويس *(يحتاج voiceStateUpdate event)* |

### فترات التجديد
| الفترة | المدة |
|--------|-------|
| `daily` | 24 ساعة |
| `2days` | 48 ساعة |
| `weekly` | 7 أيام |

### مثال عملي
```
/task setup role:@مشرف-دردشة type:messages goal:50 period:daily points:100
/task setup role:@مشرف-صوتي  type:voice    goal:3  period:weekly points:200
```
مشرف عنده الرتبتين → يرى مهمتين في `/mytasks`

## ⚖️ نظام الاستئناف — شرح تفصيلي
```
المشرف ينفذ /warn أو /timeout
        ↓
العقوبة تُنفذ فوراً + تذهب للمراجعة
        ↓
    ✅ قبول → تُضاف النقاط ✓
        ↓
    ❌ رفض → تُشال العقوبة + يحق الاستئناف
        ↓
    /appeal → استئناف أول (أي مراجع)
        ↓
    ✅ قبول → تُضاف النقاط + تُعاد العقوبة ✓
        ↓
    ❌ رفض → يحق استئناف ثانٍ وأخير
        ↓
    /appeal → استئناف ثانٍ (رتبة الاستئناف فقط)
        ↓
    القرار نهائي لا رجعة فيه
```
